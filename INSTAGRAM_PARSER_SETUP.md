# 📸 Instagram Parser Setup Guide

## 🎯 Что это делает?

Этот модуль позволяет боту парсить (загружать) посты и истории с Instagram профиля и отправлять их в Telegram чат.

**Пример использования:**
- Загружать посты из @mount.bar в Telegram группу
- Делиться контентом из Instagram с командой в Telegram
- Автоматизировать распространение контента

---

## 🔑 Настройка учетных данных

### Шаг 1: Создайте отдельный Instagram аккаунт

⚠️ **ОЧЕНЬ ВАЖНО**: Используйте отдельный аккаунт, не ваш основной!

**Почему?**
- Instagram может заблокировать аккаунт за частые запросы
- Отдельный аккаунт — это безопаснее для вашего основного профиля
- Парсинг может быть расценен как подозрительная активность

### Шаг 2: Отредактируйте `.env` файл

```bash
copy .env.example .env
```

Откройте `.env` и добавьте:

```env
# ===== INSTAGRAM PARSER =====
INSTAGRAM_USERNAME=your_account_username
INSTAGRAM_PASSWORD=your_account_password
INSTAGRAM_ACCOUNT=mount.bar
TARGET_CHAT_ID=-1003826622440
```

**Где найти эти значения:**

| Переменная | Значение | Пример |
|------------|----------|---------|
| `INSTAGRAM_USERNAME` | Ваше имя пользователя в Instagram | `my_parser_account` |
| `INSTAGRAM_PASSWORD` | Пароль аккаунта | `SecurePassword123` |
| `INSTAGRAM_ACCOUNT` | Профиль для парсинга (без @) | `mount.bar` |
| `TARGET_CHAT_ID` | ID Telegram чата для отправки | `-1003826622440` |

### Шаг 3: Получите ID Telegram чата

Нужно узнать ID группы, куда отправлять посты/истории.

**Способ 1: Через бота**
1. Добавьте бота в группу
2. Напишите сообщение в группу
3. Используйте `@userinfobot` для получения ID группы

**Способ 2: Первичный запуск**
1. Добавьте бота в группу
2. Запустите команду `/parse_posts`
3. Проверьте логи бота для ID группы

## 📬 Автоматическое чтение кода из Gmail

Если Instagram запрашивает код подтверждения, бот может попробовать автоматически взять код из вашей почты Gmail.

1. Создайте App Password в Google (если у вас включена двухфакторная аутентификация). Инструкция: https://support.google.com/accounts/answer/185833
2. В `.env` добавьте:

```env
GMAIL_EMAIL=your.email@gmail.com
GMAIL_APP_PASSWORD=your_app_password
```

3. Бот будет сканировать последние письма в `INBOX` и искать числовой код (4-8 цифр). Если код найден, он попытается автоматически применить его для входа в Instagram.

Ограничения:
- Работает через IMAP (`imap.gmail.com`), поэтому ваш Gmail должен поддерживать вход по паролям приложений или IMAP-доступ.
- OAuth2 вариант (через Gmail API) пока не реализован — могу добавить при необходимости.

## 🛠️ Управление через бота (админ)

Добавлены команды для администрирования настроек парсинга прямо из бота (доступно только администратору):

- `/ig_settings` — показать текущие настройки парсинга
- `/set_ig_account <имя>` — задать аккаунт для парсинга (без `@`)
- `/set_ig_interval <секунды>` — задать интервал проверки в секундах
- `/set_target_chat <id>` — задать ID целевого Telegram чата
- `/set_gmail_email <email>` — задать Gmail для автоподтверждений
- `/set_gmail_pass <app_password>` — задать App Password для Gmail
- `/toggle_ig_monitor` — включить/выключить фоновый монитор

После изменения настроек рекомендуется перезапустить бота, чтобы они вступили в силу.


## 🚀 Использование

### Команда 1: Парсинг постов

```
/parse_posts [количество]
```

**Параметры:**
- `количество` — опционально, количество постов (макс. 50, по умолчанию 5)

**Примеры:**

```
/parse_posts          # Загружает последние 5 постов
/parse_posts 10       # Загружает последние 10 постов
/parse_posts 20       # Загружает последние 20 постов
```

**Что происходит:**
1. Бот подключается к Instagram с вашими учетными данными
2. Загружает указанное количество постов с профиля `@mount.bar`
3. Извлекает изображения/видео, описание, лайки, комментарии
4. Отправляет все в Telegram чат `-1003826622440`

### Команда 2: Парсинг историй

```
/parse_stories
```

**Параметры:** нет

**Что происходит:**
1. Загружает все активные истории профиля
2. Отправляет их в Telegram чат
3. Истории удаляются после 24 часов на Instagram

## ⚠️ Важные замечания

### Ограничения Instagram

- Instagram может потребовать 2FA (двухфакторную аутентификацию)
- Если видите ошибку "checkpoint", Instagram запросил верификацию
- Может потребоваться ввести код из SMS или письма

### Безопасность

- **Никогда** не коммитьте `.env` с реальными паролями в Git
- Используйте `.gitignore` для исключения `.env`:
  ```
  .env
  *.pyc
  __pycache__/
  ```

### Производительность

- Загрузка больших медиа может занять время
- Посты отправляются с 1-секундной задержкой между ними
- Не рекомендуется парсить более 50 постов за раз

## 🐛 Решение проблем

### Проблема: "Ошибка входа в Instagram"

**Решения:**
1. Проверьте логин и пароль в `.env`
2. Убедитесь, что аккаунт не заблокирован
3. Попробуйте войти в Instagram вручную (может быть нужна верификация)
4. Используйте двухфакторную аутентификацию

### Проблема: "Библиотека instagrapi не установлена"

```bash
pip install instagrapi==2.0.0
```

### Проблема: "TARGET_CHAT_ID не настроен"

1. Откройте `.env`
2. Проверьте переменную `TARGET_CHAT_ID`
3. Убедитесь, что это целое число (например: `-1003826622440`)

### Проблема: "Не удалось загрузить посты"

**Возможные причины:**
- Аккаунт приватный и нет подписки
- Профиль заблокирован или удален
- Instagram ограничил доступ (слишком много запросов)
- Нет интернета

## 🔁 Автопроверка (фоновой монитор)

Вы можете включить автоматическую проверку аккаунта в фоне. По умолчанию интервал проверки задается в секундах переменной окружения `INSTAGRAM_POLL_INTERVAL`.

Добавьте в `.env`:

```env
INSTAGRAM_POLL_INTERVAL=10
```

- Значение `10` означает проверку каждые 10 секунд.
- Будьте осторожны: частые запросы (особенно < 60 секунд) увеличивают риск блокировки Instagram аккаунта.

Рекомендуется использовать интервал не меньше 30–60 секунд для стабильной работы в продакшене.

## ❓ FAQ

**Q: Может ли это заблокировать мой аккаунт?**
A: Если вы используете отдельный аккаунт и не парсите постоянно, риск минимален. Instagram имеет лимиты на запросы.

**Q: Как часто я могу парсить?**
A: Рекомендуется не более 1 раза в час. Частый парсинг может привести к ограничениям.

**Q: Можно ли парсить приватные аккаунты?**
A: Только если вы подписаны на них с используемого аккаунта.

**Q: Что делать при ошибке "checkpoint"?**
A: Это означает, что Instagram запросил верификацию. Войдите в Instagram вручную с использованного аккаунта и следуйте инструкциям.
